# 📋 Homework: ERD & Relational Schema Design (FU-02)

## 📘 Purpose

Students choose their own real-world system, then:

1. Design an **ERD** 
3. Convert the ERD to a **relational schema** (tables, primary keys, foreign keys)  

No fixed scenario is given — each student proposes their own topic with group. 
One group have to do for one schema.

---

## 🎯 Learning objectives

- Identify **entities**, **attributes**, and **relationships** from a problem description  
- Draw an ERD with correct **cardinality** (1:1, 1:N, M:N)  
- Build a **relational schema** with PK and FK  

---

## 🛠️ Tools

Students may use:

- [draw.io / diagrams.net](https://app.diagrams.net/)  
- [ERDPlus](https://erdplus.com/)  
- [dbdiagram.io](https://dbdiagram.io/)  
- Lucidchart, MySQL Workbench (EER), or similar  

See `Reference_ERD_and_Relational_Schema.md` for how to use these tools.

---

## 📌 Instructions & topic ideas

1. Pick **one** system you understand (eg. school, shop, clinic, library, hotel, etc.).  
2. Your design must include **at least 4 entities** and **at least one many-to-many (M:N)** relationship.  
3. Every entity must have a **primary key**; every relationship must appear correctly in the relational schema.  
4. Each group have done an **ERD image** (PNG or PDF) and **Relational Schema** and must be different with group.

**Topic ideas (choose one or create your own):**

| System | Possible entities |
|--------|-------------------|
| Online course platform | Student, Course, Instructor, Enrollment, Payment |
| Clinic / hospital | Patient, Doctor, Appointment, Department, Prescription |
| Book rental shop | Customer, Book, Rental, Staff, Branch |
| Restaurant | Customer, Order, MenuItem, OrderLine, Table |
| Gym | Member, MembershipPlan, Trainer, Session, Payment |
| Hotel | Guest, Room, Reservation, Staff, Invoice |
| HR Management System | Employee, Department, OT, Salary |
