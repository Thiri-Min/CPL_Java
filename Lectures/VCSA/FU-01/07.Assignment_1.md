## Git Assignment 1_Opt1: Git Workflow Management

### 1. Exercise 1
Barem: a – 30%, b – 30%

Objective: K4SD (Understand basic Git knowledge: local/remote repositories, commit conventions, branching, merging, conflict resolution)

Problem Description:  
A software development team needs to manage source code using Git. The tasks include:

- Initializing a local repository and pushing it to a remote.

Following commit message conventions.

- Creating and merging branches.

- Explaining and resolving merge conflicts.

Questions to answer:

### a) Create Local Repo and Push Remote

- Initialize a local repository (git init).

- Add a file (e.g., hello.txt) and commit.

- Connect to a remote repository (git remote add origin <URL>).

- Push changes (git push -u origin main).

- Outcome: Screenshot of terminal showing repo creation and push result.

### b) Commit Message Convention

- Make at least 3 commits using proper prefixes (feat, fix, docs).

- Outcome: Screenshot of git log --oneline showing commit history.

- Estimated Time to complete: 60 mins

### 2. Exercise 2
Barem: 40%

Objective: K4SD (Understand Git branching, merging, and conflict resolution)

Problem Description:  
Developers often work on separate branches and merge changes into the main branch. This exercise demonstrates branch creation, merging, and handling conflicts.

Questions to answer:

### a) Create Branch & Merge

- Create a new branch (git checkout -b feature-branch).

- Modify a file and commit changes.

- Merge branch into main (git merge feature-branch).

- Outcome: Screenshot of branch creation and merge process.

### b) Explain Merge Conflict

- Create two branches (branchA, branchB).

- Edit the same line in a file differently in each branch.

- Attempt to merge (git merge branchB while on branchA).

Outcome:

- Screenshot of merge conflict message in terminal.

- Written explanation of what caused the conflict and how to resolve it.

Estimated Time to complete: 60 mins
