## Git Assignment 2: Git Workflow Management

### 1. Exercise 1
Barem: a – 30%, b – 30%

Objective: (Understand basic Git knowledge: local/remote repositories, commit conventions, branching, merging, conflict resolution)

Problem Description:  
A 6-member software development team needs to manage source code using Git while simulating Agile practices. The tasks include:

- Initializing a local repository and pushing it to a remote.
- Following commit message conventions.
- Creating and merging branches.
- Explaining and resolving merge conflicts.
- Managing a mini backlog (3–5 user stories).

---

#### a) Create Local Repo and Push Remote
- **Member 1 (PO)**: Create a mini backlog file (`backlog.md`) with 3–5 user stories in the format:  
  *As a [role], I want [feature], so that [benefit].*
- **Member 2 (Scrum Master)**: Initialize a local repository (`git init`).
- Add backlog file and commit.
- Connect to a remote repository (`git remote add origin <URL>`).
- Push changes (`git push -u origin main`).
- **Outcome**: Screenshot of terminal showing repo creation and push result.

---

#### b) Commit Message Convention
- **Members 3–5 (Developers A, B, C)**:  
  - Each implements one user story in a separate branch.  
  - Make at least 3 commits using proper prefixes (`feat`, `fix`, `docs`).  
- **Outcome**: Screenshot of `git log --oneline` showing commit history with conventions.
- **Estimated Time to complete**: 60 mins

---

### 2. Exercise 2
Barem: 40%

Objective: K4SD (Understand Git branching, merging, and conflict resolution)

Problem Description:  
Developers often work on separate branches and merge changes into the main branch. This exercise demonstrates branch creation, merging, handling conflicts, and code review.

---

#### a) Create Branch & Merge
- Each developer creates a branch named after their backlog story (e.g., `feature/US-101-login`).
- Modify files and commit changes.
- Open a Pull Request (PR) linking to the backlog story.
- **Member 6 (Reviewer)**: Reviews PRs, checks commit conventions, and approves merges.
- Merge branch into `main` (`git merge feature-branch`).
- **Outcome**: Screenshot of branch creation, PR, and merge process.

---

#### b) Explain Merge Conflict
- Create two branches (`branchA`, `branchB`).
- Edit the same line in a file differently in each branch.
- Attempt to merge (`git merge branchB` while on `branchA`).
- **Outcome**:  
  - Screenshot of merge conflict message in terminal.  
  - Written explanation of what caused the conflict and how to resolve it (manual edit + commit).  
- **Estimated Time to complete**: 60 mins

---
