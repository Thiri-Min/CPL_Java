# Version Control & Agile in Project Work

## 1. Introduction
Modern software development requires both **technical discipline** and **collaborative practices**.  
Two essential components are:
- **Version Control Systems (VCS)**: Tools like Git that track changes in code.
- **Agile Methodology**: A flexible, iterative approach to project management.

When integrated into project work, these practices ensure students learn not only how to code, but also how to collaborate effectively.

---

## 2. Version Control Basics
- **Definition**: A system that records changes to files over time.
- **Key Features**:
  - Track history of code changes.
  - Collaborate without overwriting each other’s work.
  - Roll back to previous versions if needed.
- **Common Tools**: Git, GitHub, GitLab, Bitbucket.
- **Core Concepts**:
  - Repository (repo)
  - Commit
  - Branch
  - Merge
  - Pull Request


## Key Git Concepts

### Repository (Repo)
A **repository** is the core storage unit in Git.  
- It contains all project files, history, and metadata.  
- Can be **local** (on your machine) or **remote** (on platforms like GitHub, GitLab, Bitbucket).  
- Think of it as a database of your project’s evolution.  
- Example: A team creates a repo on GitHub called `web-app`. Developers clone it locally to work on features.

---

### Commit
A **commit** is a snapshot of your project at a specific point in time.  
- Each commit has a unique hash (ID).  
- Includes metadata: author, timestamp, commit message.  
- Commits form the backbone of Git’s history.  
- Example: After fixing a bug in `login.js`, you run:
  ```bash
  git add login.js
  git commit -m "Fix login validation bug"
----

### Branch
A branch is a movable pointer to a commit, used for parallel development.

Default branch is usually main or master.

Developers create branches for features, fixes, or experiments.

Branches allow isolated work without affecting the main codebase.

Example:
```
git checkout -b feature-auth
```

-----

### Merge
Merging integrates changes from one branch into another.

Preserves history by combining commits.

Can be fast-forward (simple pointer move) or no-ff (explicit merge commit).

May cause conflicts if the same code lines were modified differently.

Example:
```
git checkout main
git merge feature-auth
```

### Pull Request (PR)
A Pull Request (on platforms like GitHub/GitLab) is a request to merge changes from one branch into another.

Provides a space for code review, discussion, and automated checks.

Encourages collaboration and quality control.

Example workflow:

- Developer pushes feature-auth branch to GitHub.

- Opens a PR to merge into main.

- Team reviews code, suggests changes, and approves.

- Once approved, the PR is merged.

## 3. Agile Fundamentals
- **Definition**: A project management methodology emphasizing adaptability and collaboration.
- **Core Principles**:
  - Deliver working software frequently.
  - Respond to change over following a rigid plan.
  - Encourage collaboration between developers and stakeholders.
- **Practices**:
  - Sprints (short development cycles)
  - Daily stand-ups
  - Backlog management
  - Retrospectives

---

## 4. Integration into Project Work
- **Early Introduction**: Students learn Git basics before starting the project.
- **Continuous Use**:
  - Every code change is committed and pushed to a shared repository.
  - Branching strategies (feature branches, main branch) are applied.
- **Agile in Action**:
  - Project divided into sprints.
  - Students hold stand-up meetings to discuss progress and blockers.
  - Backlog items (features, bugs) tracked in tools like Jira, Trello, or GitHub Projects.
- **Collaboration**:
  - Pull requests reviewed by peers.
  - Agile ceremonies (planning, retrospectives) embedded into project milestones.

---

## 5. Benefits for Students
- **Practical Skills**: Learn industry-standard tools (Git, Agile boards).
- **Teamwork**: Practice collaboration and communication.
- **Adaptability**: Gain experience in iterative development.
- **Professional Readiness**: Prepare for real-world software engineering environments.

## 6. Summary
By integrating **Version Control** and **Agile** into project work:
