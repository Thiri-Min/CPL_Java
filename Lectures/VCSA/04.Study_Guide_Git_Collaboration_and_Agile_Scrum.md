# Self-Study Guide: Software Engineering Practices

This guide helps you learn modern software engineering practices step by step.  
Each section includes **study instructions**, **websites**, **YouTube videos**, and **case studies**.

---

## 1. Git Flow vs Trunk-based Development
### How to Study
- Read about both branching strategies.
- Compare pros/cons and when to use each.
- Practice by setting up repositories with both models.

### Websites
- [Atlassian Git Flow Guide](https://www.atlassian.com/git/tutorials/comparing-workflows/gitflow-workflow)
- [Trunk-Based Development](https://trunkbaseddevelopment.com/)

### YouTube
- [Git Flow Explained](https://www.youtube.com/watch?v=1SXpE08hvGs)
- [Trunk-Based Development Overview](https://www.youtube.com/watch?v=UrdwY6uN5mM)

### Case Study
- Try managing a small project with Git Flow.
- Repeat with trunk-based, compare release speed and merge conflicts.

## 2. Pull Request / Merge Request
### How to Study
- Learn the purpose of PR/MR in collaborative coding.
- Practice creating PRs on GitHub/GitLab.

### Websites

- [GitLab Merge Requests](https://docs.gitlab.com/ee/user/project/merge_requests/)

### YouTube
- [Pull Requests in GitHub](https://www.youtube.com/watch?v=rgbCcBNZcdQ)

### Case Study
- Contribute to an open-source repo by submitting a PR.
