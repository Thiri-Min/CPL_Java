---

## 🔄 Git Workflow
Daily work in Git involves branching, merging, and managing code changes effectively.

---

## 🌿 Git Flow

### Branch Types
- **main/master** – reserved for production releases  
- **develop** – integration branch for features  
- **feature** – used for new functionality  
- **release** – prepares production release  
- **hotfixes** – urgent fixes for production  

### Key Points
- Provides a well‑defined standard workflow  
- Handles complexity of features and hotfixes  
- Ensures production stability  

---

## 🔀 Git Flow – Merge
- Merges integrate feature or hotfix branches back into `develop` or `main`  
- Helps consolidate parallel work streams  

---

## 🐙 GitHub Flow
- Clean and straightforward workflow  
- Suggested by Atlassian to use [rebase](ca://s?q=Git_rebase)  
- Limitations: not suitable for multi‑deployments, multi‑environments, multi‑releases, or multi‑integrations  

---

## 🔧 Other Flows

### Environment Branches
- Separate branches for different environments (e.g., staging, testing, production)  

### Release Branches
- Dedicated branches for preparing and stabilizing releases  

### Trunk‑Based Flow
- Developers commit directly to trunk (main branch)  
- Encourages continuous integration and delivery
