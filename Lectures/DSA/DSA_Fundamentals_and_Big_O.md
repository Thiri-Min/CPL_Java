# DSA Fundamentals & Big-O

## Data Structure (DS), Abstract Data Type (ADT) and Algorithm
- **Data Structure (DS):** A way of organizing and storing data for efficient access and modification.
- **Abstract Data Type (ADT):** A theoretical model defining data and operations without specifying implementation (e.g., Stack, Queue).
- **Algorithm:** A step-by-step procedure to solve a problem or perform a computation.

**Relationship:**  
- ADTs define *what* operations are possible.  
- DS provides the *implementation*.  
- Algorithms use DS/ADT to solve problems efficiently.

---

## Linear vs Non-linear
- **Linear DS:** Data elements arranged sequentially (Array, Linked List, Stack, Queue).
- **Non-linear DS:** Data elements connected hierarchically or graph-like (Tree, Graph).

---

## Time & Space Complexity
- **Time Complexity:** Measures execution time relative to input size.
- **Space Complexity:** Measures memory usage relative to input size.
- Both are critical for analyzing efficiency.

---

## Big-O Notations
- **Definition:** Describes the upper bound of algorithm growth rate.
- **Common Notations:**
  - O(1): Constant time
  - O(log n): Logarithmic time
  - O(n): Linear time
  - O(n²): Quadratic time
- **Example:**  
  - Linear Search → O(n)  
  - Binary Search → O(log n)
